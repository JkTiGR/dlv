## 📱 PWA / QR для клиента

Клиентская схема для QR:

```text
QR -> DRAGON.com
```

- `index.html` — DRAGON.com / QR page с кнопками открытия и установки.
- `HOME.html` — выбор точки: DRAGON, DLV, Asian Kitchen, Shaurma.
- `menu.html` — меню / Web App после выбора точки.
- `manifest.json` — PWA-настройки, `start_url` ведёт сразу на `HOME.html`.
- `sw.js` — service worker и cache для клиентского shell.

После QR клиент видит:

```text
[ 🍜 Открыть меню / Web App ] -> HOME.html
[ 🤖 Скачать для Android / PWA ]
[  Скачать для iPhone / PWA ]
```

На `HOME.html` клиент выбирает направление:

```text
DRAGON
DLV
Asian Kitchen
Shaurma
```

Android: открыть QR в Google Chrome, нажать “Скачать для Android / PWA”, подтвердить установку на главный экран.

iPhone: открыть QR в Safari, нажать “Скачать для iPhone / PWA”, затем “Поделиться” -> “На экран Домой”.

После установки иконка DRAGON открывает сразу `HOME.html`, чтобы клиент выбрал нужную точку.

## 📋 ОСНОВНЫЕ ФАЙЛЫ

### 1. **cloud_sync.py** ⭐ ГЛАВНЫЙ МОДУЛЬ
**Что это:** Python класс для синхронизации с Supabase
**Что делает:**
- Подключается к облаку (Supabase)
- Создаёт/обновляет/удаляет заказы
- Синхронизирует доступность меню
- Получает статистику
- Мигрирует старые локальные данные

**Как использовать:**
```bash
python3 cloud_sync.py  # Тест подключения
```

**Используется:** из `dragon_local_server_cloud.py` и в скриптах

---

### 2. **dragon_local_server_cloud.py** ⭐ ОБЛАЧНЫЙ СЕРВЕР
**Что это:** Новая версия локального сервера с облачной синхронизацией
**Что делает:**
- Слушает на порту 8000
- Сохраняет заказы локально И в облаке
- Предоставляет REST API
- Работает как раньше, но синхронизирует с облаком

**Как запустить:**
```bash
python3 dragon_local_server_cloud.py --port 8000
```

**Возможности:**
- GET  /api/orders - все заказы
- POST /api/orders - создать заказ
- GET  /api/stats - статистика из облака
- POST /api/menu-availability - управление доступностью меню

---

### 3. **MONITOR_CLOUD.html** ⭐ ДАШБОРД МОНИТОРИНГА
**Что это:** Веб-интерфейс для просмотра в реальном времени
**Что показывает:**
- 📦 Всего заказов
- ⏳ Ожидающие заказы
- ✅ Завершённые заказы  
- 💰 Выручка
- 📊 Статистика за день

**Как открыть:**

**Обновляется:** каждые 5 секунд автоматически

### 9. **requirements_cloud.txt** - PYTHON ЗАВИСИМОСТИ


---

### 10. **.env** - КОНФИГУРАЦИЯ (СОЗДАТЬ ВРУЧНУЮ)
**Нужно создать файл .env:**
```env
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your-api-key
```

**Где получить:**
1. https://supabase.com → ваш проект
2. Settings → API
3. Copy Project URL и Anon Key

**⚠️ ВАЖНО:** Не делитесь этим файлом!

---

### 11. **supabase_schema.sql** - СХЕМА БД
**Содержит:**
- Таблица orders (заказы)
- Таблица menu_availability (доступность меню)
- Таблица order_history (история)
- Индексы
- Row Level Security

**Как использовать:**
1. Supabase → SQL Editor → New Query
2. Скопировать содержимое этого файла
3. Нажать Run
