# DRAGON Local LAN Sync

Этот режим нужен, если касса, телефон, планшет и ноутбук должны видеть одни и те же заказы внутри одной локальной сети.

Как это работает:

- один ноутбук запускает `dragon_local_server.py`;
- все страницы открываются с этого ноутбука по его IP;
- заказы сохраняются в общей локальной базе `dragon_local_orders.json`;
- `MONITOR.html`, `VIEW.html`, `profil.html`, `contron.html` и `DRAGON_KASSA.html` получают одинаковые данные через bridge API;
- обновления приходят почти сразу через `EventSource` (`/api/local-events`).

## Быстрый старт

1. Подключи телефон, планшет и ноутбук к одной Wi-Fi сети.
2. На главном ноутбуке открой терминал в папке проекта:

```bash
cd /home/jkbook/DRAGON
bash start_dragon_lan.sh
```

3. Скрипт покажет готовые ссылки вида:

```text
http://192.168.1.50:8000/DRAGON_KASSA.html
http://192.168.1.50:8000/MONITOR.html
http://192.168.1.50:8000/index.html
```

4. Открой эти ссылки на нужных устройствах:

- `DRAGON_KASSA.html` - касса
- `index.html` - клиентская страница заказа
- `MONITOR.html` - кухня / монитор
- `VIEW.html` - клиентский просмотр
- `profil.html` - профиль / история
- `contron.html` - control panel

## Что уже синхронизируется

- новые заказы;
- статусы `NEW / COOKING / READY / HANDED / CANCELLED`;
- очистка заказов;
- доступность меню;
- локальные live-обновления по SSE.

## Проверка

После запуска можно проверить API вручную:

```bash
curl http://127.0.0.1:8000/api/health
curl http://127.0.0.1:8000/api/local-orders
curl http://127.0.0.1:8000/api/stats
```

Если всё нормально, `api/health` вернёт `"ok": true`.

## Практический сценарий

1. Клиент открывает `index.html` на телефоне и создаёт заказ.
2. Касса видит тот же заказ в `DRAGON_KASSA.html`.
3. Кухня видит его в `MONITOR.html`.
4. После смены статуса обновление появляется на других устройствах без ручного refresh.

## Если телефон не открывает страницу

- убедись, что все устройства в одной сети;
- проверь IP ноутбука командой `hostname -I`;
- проверь, что порт `8000` не блокируется firewall;
- попробуй открыть `http://IP_НОУТБУКА:8000/api/health` прямо с телефона.
